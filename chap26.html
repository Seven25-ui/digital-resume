Chapter 26: API Integration in Termux

A System Architect doesn't work in isolation. In this chapter, we will learn how to connect our scripts to the outside world using REST APIs. We will focus on using Python's requests library to fetch and send data.

Concept: Your Script (Client) ➔ HTTP Request ➔ API Server ➔ JSON Response ➔ Your Script

1. Installing the Connectivity Tools

We need the requests module, which is the industry standard for HTTP in Python:

pip3 install requests

2. Building a Real-Time Data Fetcher

Let's create a script that fetches the current Bitcoin price—a classic example of API usage. Create: nano btc_price.py

import requests

def get_btc_price():
    url = "https://api.coindesk.com/v1/bpi/currentprice.json"
    response = requests.get(url)
    data = response.json()
    
    price = data['bpi']['USD']['rate']
    print(f"--- MATRIX MARKET DATA ---")
    print(f"Current BTC Price: ${price} USD")

if __name__ == "__main__":
    get_btc_price()

3. Automating Notifications

You can combine this with **Telegram's Bot API** to send yourself an alert if a certain condition is met (e.g., if a server goes down or a price drops). This is how modern monitoring systems are built.

4. JSON Parsing

Most APIs return data in JSON format. Python makes it easy to convert this into a dictionary using .json(), allowing you to access specific data points with ease.
