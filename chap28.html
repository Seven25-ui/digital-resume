Chapter 28: Automated Security Patching

In the digital world, new vulnerabilities are discovered every day. A System Architect's duty is to ensure that the Matrix is always patched. We will automate the update process so that your Debian system stays secure without manual intervention.

Why Automate Updates? Hackers often use automated bots to find unpatched systems. By automating your security updates, you close the door before they can even knock.

1. The "Unattended-Upgrades" Utility

Debian has a powerful package called unattended-upgrades. It handles the installation of security patches automatically.

sudo apt update && sudo apt install unattended-upgrades -y

2. Configuration

To enable it, you need to run the configuration wizard:

sudo dpkg-reconfigure -plow unattended-upgrades

Select "Yes" to enable automatic downloads and installations of security updates.

3. Custom Automation Script

If you prefer a custom Bash script to log the updates, you can use this:

#!/bin/bash
# Log file path
LOG_FILE="/home/loy/Digital/logs/security_patch.log"

echo "--- Security Check: $(date) ---" >> $LOG_FILE
sudo apt update && sudo apt upgrade -y >> $LOG_FILE
echo ">>> System Patched Successfully." >> $LOG_FILE

4. Scheduling the Patch

Most Architects schedule security updates during "Low Traffic" hours (e.g., 3:00 AM). Add this to your crontab -e:

0 3 * * * /bin/bash /home/loy/Digital/security_patch.sh
