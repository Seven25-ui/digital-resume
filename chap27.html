Chapter 27: Log Rotation & Management

Every automated script generates data. If you don't manage this data, your system's storage will eventually hit 100%, causing a total system failure. This is where Log Rotation comes in.

1. The Problem: "Log Bloat"

Scenario: Your backup script runs every hour and writes a 1MB log. In a year, that's nearly 9GB of just text files!

2. Using the Linux logrotate Utility

Linux has a built-in tool called logrotate. It automatically compresses, renames, or deletes old log files based on their age or size.

Example configuration for your custom Matrix logs:

/home/loy/Digital/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
}
  • daily: Rotate the log every day.
  • rotate 7: Keep only the last 7 days of logs.
  • compress: Zip the old logs to save space (.gz).

3. Manual Cleanup Script

If you want to build your own "Janitor Script" using Bash to delete logs older than 30 days:

#!/bin/bash
# Find and delete files ending in .log older than 30 days
find /home/loy/Digital/logs/ -name "*.log" -type f -mtime +30 -delete

4. Why Architects Love This

A "self-cleaning" system is a professional system. Implementing this shows that you're not just a coder, but a Reliability Engineer.
