Chapter 25: Database Automation

Data is the lifeblood of the Matrix. In this chapter, we will learn how to automate MariaDB/MySQL backups and basic database maintenance using Shell scripts and Cron.

SQL Database ➔ SQL Dump ➔ Compression (Gzip) ➔ Remote Storage

1. Setting up the Environment

Ensure you have MariaDB installed in your Debian environment:

sudo apt update && sudo apt install mariadb-server -y

2. The Automated Backup Script

Instead of manually exporting your database, we will create a script that adds a timestamp to every backup. Create: nano db_backup.sh

#!/bin/bash

# Configuration
DB_NAME="matrix_db"
BACKUP_PATH="/home/loy/backups"
DATE=$(date +%Y-%m-%d_%H%M)

# Create backup directory if not exists
mkdir -p $BACKUP_PATH

# Execute Dump
mysqldump -u root $DB_NAME > $BACKUP_PATH/$DB_NAME-$DATE.sql

# Compress
gzip $BACKUP_PATH/$DB_NAME-$DATE.sql

echo ">>> Backup of $DB_NAME completed at $DATE"

3. Automated Optimization

Databases can become slow over time. We can automate the OPTIMIZE TABLE command to keep the Matrix running smoothly:

mysqlcheck -o --all-databases -u root

4. Integration with Cron

To run this backup every night at 2:00 AM, add this to your crontab -e:

0 2 * * * /bin/bash /home/loy/Digital/db_backup.sh
