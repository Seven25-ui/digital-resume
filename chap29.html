Chapter 29: DevOps Pipeline Basics

In modern software development, we don't just "upload files." We use a Pipeline. This ensures that every piece of code is tested and verified before it goes live. This is the heart of DevOps.

1. The Concept of CI/CD

Continuous Integration (CI): Automatically testing code as soon as it is pushed to Git.

Continuous Deployment (CD): Automatically updating the live website once the tests pass.

2. Building a Mini-Pipeline

You can simulate a DevOps pipeline in Termux by creating a "Deploy Script" that only pushes to GitHub if the HTML files are valid. Create: nano deploy.sh

#!/bin/bash
echo ">>> Starting Pipeline..."

# Step 1: Check if index.html exists
if [ -f "index.html" ]; then
    echo "Check Passed: Source files found."
else
    echo "Check Failed: index.html missing!"
    exit 1
fi

# Step 2: Deploy to GitHub
git add .
git commit -m "Auto-deploy via Matrix Pipeline: $(date)"
git push origin main

echo ">>> Deployment Successful!"
