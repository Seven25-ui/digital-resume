Chapter 22: Python for System Admin

While Bash is great for simple tasks, Python excels when we need to handle complex logic, parse JSON data, or interact with APIs. In this chapter, we will use Python to monitor our system resources.

1. Installing Python in the Matrix

In your Termux-Debian environment, ensure Python is installed and up to date:

sudo apt update && sudo apt install python3 python3-pip -y

2. Building a Resource Monitor Script

We will create a script that checks the CPU load and memory usage. Create the file: nano monitor.py

import os
import shutil

# Get Disk Usage
total, used, free = shutil.disk_usage("/")

print(f"--- SYSTEM RESOURCE MONITOR ---")
print(f"Total Disk: {total // (2**30)} GB")
print(f"Used Disk: {used // (2**30)} GB")
print(f"Free Disk: {free // (2**30)} GB")

# Check CPU Load
load1, load5, load15 = os.getloadavg()
print(f"CPU Load (1m): {load1}")

3. Why use Python over Bash?

  • Readability: Python code is easier to maintain.
  • Libraries: Access to thousands of modules (like requests for web or psutil for hardware).
  • Error Handling: Better way to catch crashes and log errors.

4. Execution

To run your script, simply type:

python3 monitor.py
